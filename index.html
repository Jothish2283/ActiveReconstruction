Uncertainty-Aware Active 3D Reconstruction via Reinforcement Learning

A PPO-trained camera agent learns to select viewpoints that maximally reduce geometric uncertainty in a partially reconstructed 3D scene — achieving +790% reward improvement over random baseline within 30,000 training steps.

System Design

End-to-End Architecture

Five interacting modules form a complete active reconstruction pipeline, all implemented in Unity C# with ML-Agents Python trainer.

🎯
CameraAgent.cs
PPO agent. 144-dim observation vector. Discrete action space (64 viewpoints). Shaped reward function.
🗺️
ReconstructionManager.cs
32³ voxel occupancy grid. Exponential confidence update via stratified raycasting. Ground truth via OverlapBox.
🌐
ViewPointGenerator.cs
Fibonacci sphere lattice. 64 deterministic viewpoints. Uniform angular distribution with golden angle spacing.
🎨
VoxelColorAnimator.cs
Live uncertainty heatmap. Red = uncertain, green = confident. Only renders occupied voxels (ship geometry).
📊
MetricsLogger.cs
Per-episode CSV logging. Reward, coverage, uncertainty, accuracy, runtime. Auto-summary after N episodes.
🎲
Baselines
RandomViewSelector (seeded RNG) and SequentialViewSelector (fixed Fibonacci order). Both auto-stop + print summary.

Reward Function

// Shaped reward at each step rt = w₁ · Δūt // uncertainty reduction (w₁ = 1.0) + w₂ · nnew // new voxels observed (w₂ = 0.005) − w₃ · 𝟙[revisit] // revisit penalty (w₃ = 0.30) + w₄ · 𝟙[final] // terminal coverage bonus (w₄ = 0.5)

Observation Space (144 floats)

3 Agent position (normalised to [-1,1]³) 8 Uncertainty histogram (8 bins over occupied voxels) 3 Uncertainty-weighted centroid (normalised) 64 Per-viewpoint estimated uncertainty gain 64 Binary visited mask (1 = already visited) 1 Steps remaining (normalised) 1 Current mean uncertainty (scalar) ───────────────────────────────────────────── 144 Total → Space Size = 144 in BehaviorParameters

Discussion

Key Findings

+790%
Reward Improvement
RL reward (+3.48) vs random (−0.91). The agent learned to avoid revisiting viewpoints (near-zero revisit penalties) while targeting high-uncertainty equatorial views.
~63%
Geometric Coverage Ceiling
No strategy exceeds ~63% coverage due to mutual occlusion between spacecraft components. This is a scene property, motivating future work with continuous action spaces.
↓13%
Sequential Polar Bias
Sequential visits Fibonacci poles first, missing equatorial geometry. Scores 13% lower coverage than RL and even lower than random — confirming systematic strategies fail without uncertainty awareness.
32
Views from 64
With 32 steps from 64 viewpoints, the agent must choose 50% of available views. This selection pressure is what enables RL to demonstrate clear advantage over baselines.
Getting Started

Quick Setup

1 — Clone & Install

git clone https://github.com/jothish2283/ActiveReconstruction.git cd ActiveReconstruction python -m venv mlagent_env mlagent_env\Scripts\activate pip install mlagents==0.30.0 torch==1.13.1

2 — Train PPO

# Run BEFORE pressing Play in Unity mlagents-learn Config/camera_agent.yaml \ --run-id=CameraPPO_v1 --train # Then press Play in Unity Editor # Training: 30,000 steps (~15 min)

3 — Generate Figures

# Edit paths at top of script, then: python Python/plot_all.py # Outputs 7 figures (PNG + PDF) # to the figures/ directory
